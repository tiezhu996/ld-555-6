## 项目主要需求

### 一、项目概述

GGArena 是一个面向电竞爱好者的社区平台，支持战队管理、赛事组织、玩家交流和排行榜展示。本项目为纯前端应用，所有数据通过 localStorage 和 IndexedDB 进行本地持久化，WebSocket 通信通过 mock 层模拟实现。UI 采用电竞风格设计，支持暗色/亮色主题切换。

### 二、核心数据模型

#### 1. 战队 Team

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | UUID |
| name | string | 战队名称 |
| logo | string | 战队 Logo URL/base64 |
| game | GameTitle | 主要游戏（LOL/CSGO/DOTA2/VALORANT/PUBG） |
| members | string[] | 成员 Player ID 列表 |
| captainId | string | 队长 Player ID |
| rank | number | 战队积分排名 |
| wins | number | 胜场数 |
| losses | number | 负场数 |
| bio | string | 战队简介 |
| createdAt | string | 创建时间 |

**业务动作**：创建战队 → 邀请玩家 → 踢出成员 → 转让队长 → 修改战队信息 → 解散战队

#### 2. 赛事 Tournament

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | UUID |
| name | string | 赛事名称 |
| game | GameTitle | 比赛游戏 |
| format | TournamentFormat | 赛制（单败淘汰/双败淘汰/循环赛/积分赛） |
| teamSize | number | 每队人数 |
| maxTeams | number | 最大参赛队伍数 |
| startDate | string | 开始日期 |
| endDate | string | 结束日期 |
| prize | string | 奖金/奖品描述 |
| status | TournamentStatus | 状态（报名中/进行中/已结束） |
| rules | string | 赛事规则（Markdown 格式） |
| bracket | object | 赛程对阵图数据 |

**关系**：赛事有多支参赛队伍，每支队伍有多场对局记录
**业务动作**：创建赛事 → 开放报名 → 队伍报名 → 确认对阵 → 记录比赛结果 → 更新赛程图 → 颁奖结束

#### 3. 玩家 Player

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | UUID |
| username | string | 用户名 |
| avatar | string | 头像 URL/base64 |
| bio | string | 个人简介 |
| mainGame | GameTitle | 主要游戏 |
| rank | PlayerRank | 段位 |
| teams | string[] | 所属战队 ID 列表 |
| matchHistory | string[] | 对局记录 ID 列表 |
| onlineStatus | 'online' \| 'offline' \| 'in-game' | 在线状态 |

**业务动作**：注册/编辑资料 → 加入战队 → 退出战队 → 参加比赛 → 查看战绩

#### 4. 对局记录 Match

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | UUID |
| tournamentId | string | 所属赛事 ID |
| teamA | string | 队伍 A 的 ID |
| teamB | string | 队伍 B 的 ID |
| score | { a: number; b: number } | 比分 |
| winner | string | 获胜队伍 ID |
| playedAt | string | 比赛时间 |
| mvp | string | MVP 玩家 ID |

### 三、共享枚举定义

**所有枚举必须单独维护为常量文件**，`src/constants/enums.ts` 为唯一来源。README 中必须以表格列出每个枚举在项目中出现的所有文件位置（包括 store、组件、hooks、mock 数据、类型定义等）。

| 枚举名 | 值 | 使用位置说明 |
|--------|------|------------|
| GameTitle | LOL / CSGO / DOTA2 / VALORANT / PUBG | Team 模型、Tournament 模型、Player 模型、赛事筛选、战队筛选、首页推荐、GameTag 组件 |
| TournamentFormat | SINGLE_ELIM / DOUBLE_ELIM / ROUND_ROBIN / POINT_BASED | Tournament 模型、创建赛事表单、BracketChart 组件渲染逻辑 |
| TournamentStatus | REGISTRATION / IN_PROGRESS / FINISHED | Tournament 模型、赛事列表筛选、赛事卡片状态标签、报名按钮显隐控制 |
| PlayerRank | BRONZE / SILVER / GOLD / PLATINUM / DIAMOND / MASTER / GRANDMASTER | Player 模型、排行榜排序、段位徽章展示、rank-configs.ts 颜色映射 |

此外，`src/constants/game-configs.ts` 存储各游戏的基础配置（图标、颜色、默认队人数等），`src/constants/rank-configs.ts` 存储段位配置（图标、颜色、分数区间）。

### 四、核心页面

#### 1. `/` 首页

- **热门赛事轮播**：展示进行中和即将开始的赛事卡片，自动轮播 + 手动切换，消费 Tournament 实体
- **推荐战队**：按胜率/人气推荐战队卡片（使用 `TeamCard` 组件），消费 Team 实体
- **在线玩家数**：实时统计（mock WebSocket），动态更新数字 + 脉冲动画
- **快速入口**：创建赛事、创建战队、加入比赛
- **游戏分类**：按游戏类型快捷筛选入口（使用 `GameTag` 组件）

#### 2. `/tournaments` 赛事列表与详情

- **赛事列表**：卡片网格展示，支持按游戏（`GameTag` 筛选器）和状态筛选，Framer Motion 入场动画
- **赛事详情** `/tournaments/:id`：
  - 赛事信息头部（名称、游戏、赛制、奖金、状态）
  - **赛程图**（使用 `BracketChart` 组件，支持单败/双败/循环赛不同渲染）
  - 参赛队伍列表（使用 `TeamCard` 组件）
  - 对局结果列表
  - 报名操作（状态为 REGISTRATION 时显示）
- **创建赛事** `/tournaments/create`：表单支持选择赛制、设置参数、编写规则

#### 3. `/teams` 战队广场与详情

- **战队广场**：卡片网格展示所有战队（使用 `TeamCard`），支持按游戏 + 段位筛选，搜索
- **战队详情** `/teams/:id`：
  - 战队基本信息（Logo、名称、游戏、战绩）
  - 队员列表（使用 `PlayerBadge` 组件，展示头像+用户名+段位+在线状态）
  - 赛事参与历史
  - 对局记录时间线
  - 加入/退出战队操作
- **创建战队**：表单设置战队名称、游戏、简介、上传 Logo

#### 4. `/profile` 个人中心

- **个人信息**：头像、用户名、简介、段位展示（`PlayerBadge` 大尺寸）
- **我的战队**：所属战队列表，队长标识，快捷管理入口
- **参赛记录**：按时间线展示参加过的赛事和对局结果（使用 `BracketChart` 缩略版）
- **段位成就**：段位进度条 + 已获得成就徽章
- **编辑资料**：修改头像、简介、主要游戏

#### 5. `/leaderboard` 排行榜

- **战队排名**：表格展示，支持按游戏筛选，展示胜率、积分、战绩，使用 `TeamCard` 迷你版
- **个人段位排行**：表格展示，`PlayerBadge` + 段位图标 + 积分
- **赛事积分榜**：按赛事查看各队积分排名
- **标签页切换**：三个排行榜通过标签页切换

#### 共用组件

| 组件 | 被使用页面 | 说明 |
|------|-----------|------|
| TeamCard | 首页推荐、赛事详情参赛列表、战队广场、排行榜 | 战队信息卡片，展示 Logo+名称+游戏+战绩+队长 |
| BracketChart | 赛事详情赛程图、个人中心参赛记录 | 赛程对阵图，支持单败淘汰/双败淘汰/循环赛多种布局 |
| PlayerBadge | 战队详情队员列表、排行榜个人排行、个人中心 | 玩家徽章，展示头像+用户名+段位+在线状态灯 |
| GameTag | 首页游戏分类、赛事列表筛选、战队广场筛选 | 游戏标签，展示游戏图标+名称+代表色 |

### 五、横切关注点

#### 1. 实时通信（WebSocket Mock）

**实现方式**：创建 WebSocket mock 层，模拟服务器推送事件

**触达层面**：
- **WebSocket mock 层**：`src/mock/websocket.ts` 模拟连接、断开、消息推送
- **全局状态**：chatStore.ts 管理在线用户列表、未读通知数、实时比分
- **通知组件**：全局 Toast 通知，新赛事提醒、比赛结果通知、战队邀请通知

**模拟场景**：
- 玩家在线状态随机切换（30秒间隔）
- 进行中赛事比分更新（60秒间隔）
- 新赛事创建通知
- 战队邀请推送

#### 2. 主题切换

**实现方式**：CSS 变量 + localStorage 持久化

**两种主题**：
- **暗色主题（默认）**：深色背景 + 霓虹色调高亮，电竞风格
- **亮色主题**：浅色背景 + 柔和色彩

**触达层面**：
- CSS 变量定义在 `variables.scss`，所有组件通过变量引用颜色
- ThemeToggle 组件放在全局 Header
- useTheme() hook 管理主题状态，读写 localStorage

### 六、技术实现要求

- **状态管理**：Zustand，每个实体一个独立 store 文件
- **IndexedDB**：封装为统一的 `src/db/` 模块，提供 CRUD 操作 Promise 化接口
- **Mock 数据**：`src/mock/` 目录包含各实体的初始数据（至少各 10 条），以及 WebSocket 模拟逻辑
- **路由**：React Router v6，路由配置独立文件
- **动画**：Framer Motion 用于页面切换、列表入场、卡片 hover 效果
- **类型安全**：所有数据模型必须有 TypeScript 类型定义，放在 `src/types/` 目录

### 七、全局异常处理

**前端请求拦截器**：封装 IndexedDB 和 localStorage 操作的统一错误处理层，操作失败时通过 Toast 组件展示友好错误提示，不暴露技术细节给用户。所有异步操作必须有 try-catch 包裹。

## 文件结构强制清单

**严禁将多个职责合并到单一文件中。** 每个文件必须只承担一个明确的职责。例如：不能在一个组件中同时定义数据获取逻辑和渲染逻辑（应通过 hooks 分离）；不能在一个 store 中管理多个不相关的实体状态；不能在一个文件中同时定义 mock 数据和 mock 服务逻辑。

```
src/
├── stores/        # teamStore.ts, tournamentStore.ts, playerStore.ts, matchStore.ts, chatStore.ts
├── types/         # team.d.ts, tournament.d.ts, player.d.ts, match.d.ts
├── constants/     # enums.ts, game-configs.ts, rank-configs.ts
├── components/common/  # TeamCard, BracketChart, PlayerBadge, GameTag, EmptyState, ThemeToggle
├── hooks/         # useTeam(), useTournament(), usePlayer(), useWebSocket(), useTheme()
├── pages/         # Home, Tournaments, TournamentDetail, Teams, TeamDetail, Profile, Leaderboard, CreateTournament
├── router/        # index.ts, routes.ts
├── db/            # IndexedDB 封装（team-db.ts, tournament-db.ts, player-db.ts, match-db.ts, index.ts）
├── mock/          # mock-data.ts, websocket.ts, seed.ts
├── styles/        # global.scss, variables.scss, themes/
└── utils/         # format.ts, validation.ts, storage.ts
```
